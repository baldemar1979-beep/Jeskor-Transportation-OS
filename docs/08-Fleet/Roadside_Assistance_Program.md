# Roadside Assistance Program

## Purpose

This document establishes Jeskor Transportation's roadside assistance process for tractor-related events.

The objective is to protect driver safety, reduce downtime, and restore operations quickly.

## Roadside Event Examples

* Tractor breakdown
* Tire failure
* Electrical issue
* Air leak
* Warning light
* No-start condition
* Mechanical failure

## Driver Responsibilities

Drivers shall:

* Stop in a safe location when possible
* Activate hazard lights
* Contact management immediately
* Provide exact location
* Describe the issue clearly
* Remain available for updates

## Management Responsibilities

Management shall:

* Confirm driver safety
* Determine service need
* Contact appropriate vendor
* Monitor repair progress
* Communicate operational impact
* Document the event

## Vendor Selection

Use preferred vendors when available.

If preferred vendors are unavailable, management may approve alternate service providers.

## Escalation Triggers

Escalate when:

* Driver is in unsafe location
* Repair delay threatens block execution
* Towing is required
* Vendor response is delayed
* Tractor may be out of service

## Documentation

Record:

* Tractor number
* Driver name
* Location
* Issue
* Vendor contacted
* Response time
* Repair outcome
* Downtime

## Performance Metrics

Monitor:

* Vendor response time
* Repair completion time
* Roadside event frequency
* Cost per event

## Objective

Provide consistent roadside support that protects drivers and minimizes operational disruption.
